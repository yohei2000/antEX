# Decision Log

このファイルは、antEX の設計判断を記録するためのログである。

単なる実装履歴ではなく、ゲームの方向性に影響する判断と、その結果として採用または削除したものを残す。

## 2026-06-22: リポジトリを antEX として分離

### Decision

前バージョンのリポジトリとは分けて、新しく `antEX` として開発する。

### Result

- 既存の別リポジトリは前の状態に戻す対象とした
- antEX は新しい方向性の実験と実装を受け持つ
- 公開先は `https://github.com/yohei2000/antEX`

## 2026-06-22: メインコンセプトを放置ゲームへ変更

### Decision

antEX の中心コンセプトを「アリの巣の大帝国を目指す放置ゲーム」にする。

### Result

- 水、枝、物、餌を置いて反応を見るラボ型機能は主軸から外した
- 時間経過による食料増加、アリ増加、巣の成長を中心にした
- プレイヤーの主な判断は、強化と遠征に寄せる

## 2026-06-22: 初期アリ数を少なくする

### Decision

最初から大量のアリを出さない。

### Result

- 初期アリ数は 12 匹程度にした
- 盤面上の表示数には上限を設ける
- 小さな巣から始まる成長感を優先する

## 2026-06-22: アリの見た目を小さくする

### Decision

アリが大きすぎて不快に見える状態を避ける。

### Result

- アリの表示スケールを控えめにする
- 画面上では個体の細部より、群れとしての動きを重視する
- 頭、胸、腹、触角、6 本脚のシルエットは維持する

## 2026-06-22: マップを広げ、自然環境を入れる

### Decision

巣の周辺だけでなく、広めの地形と複数の餌場を持つ自然環境にする。

### Result

- ワールド半径を広めに設定した
- 複数の餌場を配置した
- 地形パッチによって盤面に変化を出した
- 採餌ルートと危険地帯が生まれる方向にした

## 2026-06-22: 水や枝の配置ツールを削除

### Decision

水、枝、物などを置いてアリの反応を見る機能は削除する。

### Result

- 下部ツールバー型の観察ラボ操作は廃止した
- UI は成長、強化、遠征に寄せた
- プレイヤーの目的を「反応を見る」から「巣を育てる」に明確化した

## 2026-06-22: 外敵として敵対アリを追加

### Decision

外敵は、見た目の違う少数のアリとして配置する。

### Result

- 敵対アリは味方より少し大きい
- 色は茶色系にする
- ランダムに配置される
- 餌場に寄りやすく、味方の採餌を邪魔する

## 2026-06-22: 接触時は個体パラメータで押し合う

### Decision

味方アリと敵対アリが接触した場合、単純な固定勝敗ではなく、個体パラメータで優先される側が相手を弾く。

### Result

- 味方側は役割、慎重さ、粘り強さを使う
- 敵対側は攻撃性、しぶとさ、体格を使う
- 負けた味方アリはパニック化し、採餌行動を中断される
- 外敵は盤面上の採餌効率を下げる環境要素になった

## 2026-06-22: 敵アリ戦闘を即時ノックバックから短い交戦へ変更

### Decision

敵アリとの接触は一瞬で勝敗を出さず、約2秒の組み合いを見せてから、個体パラメータに基づいて敗者が巣またはホーム地点へ逃げ帰る形にする。

### Result

- 勝敗計算の根拠は既存の役割、粘り強さ、警戒心、敵アリの攻撃性を維持する
- 大きな即時押し出しをなくし、接触直後は小さな位置調整だけにする
- 敵アリ色は明るい橙から低彩度の赤茶へ変更し、自然環境内で浮きすぎないようにする
- スマホでは管理パネルをスワイプで折り畳み、アリの動きを見られる面積を確保する

## 2026-06-23: 喧嘩中の回転を抑え、カメラズームを追加

### Decision

敵アリとの交戦中は、アンカー地点の周囲をくるくる回るのではなく、その場で組み合っているように見せる。観察性を上げるため、スマホは二本指ピンチ、PCはマウスホイールでカメラ距離を変えられるようにする。

### Result

- 交戦中の位置表現は固定した組み合い軸と小さな揺れに寄せる
- 勝敗計算、交戦時間、敗者逃走、セーブデータ形式は変更しない
- カメラズームは既存のカメラ距離だけを変え、盤面のゲーム進行には影響させない
- Playwright と verify で、喧嘩中の中心位置・向きの変化とズーム操作を確認する

## 2026-06-23: 巣の成長強化をツリー化

### Decision

強化を単純な6項目から、実際のアリの巣で観察される機能に近い枝へ分ける。モデル化する軸は、採餌網、育房、巣構造、防衛の4系統にする。

### Basis

- 働き蟻は採餌、育児、巣作り、防衛を担うため、強化もその社会機能へ対応させる
- 幼虫と蛹は安定した温度の育房に移されるため、育房と保温を孵化速度へ対応させる
- 採餌は帰巣する個体の接触や匂い道で調整されるため、採餌道と匂い道を採餌効率へ対応させる
- 巣は地下の坑道、部屋、換気、廃棄場所を持つため、坑道分岐、換気孔、廃棄坑を収容上限、回復、防御へ対応させる

Sources:
- https://en.wikipedia.org/wiki/Ant
- https://en.wikipedia.org/wiki/Red_harvester_ant
- https://arxiv.org/abs/2107.11075
- https://press.princeton.edu/books/hardcover/9780691179315/ant-architecture

### Result

- 強化ノードを13個に増やした
- 旧強化IDは維持し、新しい強化IDはセーブ読み込み時に0で補う
- 深いノードは前提Lvを必要にし、序盤から全効果を一気に買えないようにした
- 最大強化時でも採餌、孵化、収容、防衛が極端に跳ねない範囲をPlaywrightとverifyで検証する

## 2026-06-23: 巣の入口と自然地形を強化

### Decision

巣は平たい黒丸ではなく、斜面に開いた入口、土の縁、掘り出し土で表現する。フィールドは色パッチだけにせず、低い土塊、自然石、細い枝を置いて、採餌を完全に妨げない範囲で地面の凹凸と障害物を読む体験を足す。

### Result

- 巣入口を 4 か所にし、入口ごとに暗い穴、縁、掘り出し土を持たせた
- 巣レベル成長時は入口と掘り出し土も基準値から再配置し、スケールがフレームごとに累積しないようにした
- フィールドに低い土塊 10、自然石 6、枝 5 を固定配置した
- 自然石は衝撃リングを持たない静的障害物とし、既存のプレイヤー操作由来の石とは表示更新を分けた
- セーブデータ形式と強化/戦闘/ズーム仕様は変更しない

## 2026-06-23: 遠征戦闘を個体エージェント物理へ移行開始

### Decision

遠征戦闘の勝敗を、旧来の戦力比率と乱数による勝率抽選から切り離し、個体アリの接触、向き、押し合い、噛みつき、士気低下、撤退から決める agent engine へ移す。既存の通常盤面と保存形式は変更しない。

### Result

- `src/expedition/agent/` に deterministic RNG、spatial hash、AntAgent 型、固定ステップ個体物理、状態機械、metrics、adapter、InstancedMesh renderer の基礎を追加した
- `startExpedition()` は `runExpeditionAgentBattle()` の結果を使い、成功/失敗、負傷、報酬、脅威変化、診断を agent battle 由来にした
- 遠征UIの勝率表示は確率値ではなく、個体戦エンジンの表示に変更した
- 遠征開始時に agent battle の frame log を replay として通常3D scene上に InstancedMesh 表示する
- `agent` を既定の遠征engine設定として明示し、旧群モデルは `src/expedition/legacy/` に隔離して `expeditionEngine=legacy` の場合だけ使用する
- `recoil` と `disengage` を噛みつき後の反動・離脱状態として物理更新に組み込んだ
- Unit test で deterministic、停止時 jitter、heading alignment、turn limit、contact、retreat、pheromone/objective、collision、integration、ant-likeness thresholds を検証した
- Playwright/verify で agent battle の生成、物理ログ、replay agent 表示、診断ログを検証した

## 2026-06-24: 遠征戦闘を既存アリ個体へ接続

### Decision

遠征開始時に別 renderer の戦闘用アリを重ねるのではなく、通常画面にいる既存 `Ant3D` を選抜し、その id、位置、向き、歩行位相、描画slotを agent battle の初期状態へ渡す。戦闘 frame log は同じ `Ant3D` へ適用し、通常 `AntRenderSystem` で描画する。

### Result

- agent battle は `playerSeeds` を受け取り、既存個体の id/position/heading/gaitPhase/renderIndex/spawnReason を初期frameに保持する
- `AntRenderSystem` は id 由来の stable slot を持ち、戦闘中に不要な render index swap を起こさない
- 遠征中の参加個体は通常更新を止め、agent frame log で同じインスタンスを制御し、終了時に負傷/撤退/復帰を同じ個体へ反映する
- 追加の敵個体は通常 renderer に流す軽量 visual とし、別 expedition renderer は生成しない
- `src/expedition/qa/AntBattleInspector.ts` を追加し、identity/teleport/二重描画/状態異常/perf 兆候を simulation へ副作用なく report する
- Unit/Playwright/verify で transform continuity、renderIndex 継続、Inspector 診断、legacy flag の分離を検証する

## 2026-06-24: 遠征挙動だけを確認するモードを追加

### Decision

通常プレイや保存データを変えずに遠征挙動だけを確認できるよう、URL パラメータ `?expeditionOnly=1` を確認モードとして扱う。確認モードでは保存データを読まず、localStorage にも書かず、一時的なコロニー状態で遠征タブを開き、自動で agent 遠征を開始する。

### Result

- `?expeditionOnly=1` または `?mode=expedition` で遠征確認モードに入る
- 成長タブと資源バーを隠し、遠征タブと戦闘ログだけを前面に出す
- 遠征終了後は短い間隔で次の遠征を再開できるようにし、連続観察しやすくした
- 確認モードでは `saveColony()` を無効化し、既存セーブデータを汚さない
- Playwright で自動開始、参加個体、敵visual、critical diagnosticsなし、保存非書き込みを検証する

## 2026-06-24: 敵アリを常駐から予告付き定期襲撃へ変更

### Decision

敵アリが巣の周辺や餌場に最初から常駐していると、プレイヤーから見て「なぜそこにいるか」が読みにくい。通常盤面の敵アリは、防衛ゲーム寄りの定期襲撃として扱い、襲撃前に予兆を出してから外縁から侵入させる。

### Result

- 初期状態と襲撃待機中は通常盤面の敵アリを0匹にした
- `raidState` をセーブデータへ optional 追加し、旧セーブでは待機状態を補完する
- 保存中に active/retreating だった襲撃は、再読み込み時に warning へ戻し、敵が突然近くに出る状態を避ける
- 襲撃は `calm -> warning -> active -> retreating/recovering -> calm` のサイクルで進む
- warning では外縁に警戒フェロモンとログを出し、active で最大4匹が外縁から侵入する
- 敵脅威は襲撃間隔、敵数、襲撃敵の強さへ小さく反映する
- 巣の防衛値は交戦時の味方防衛力と襲撃間隔へ反映し、防衛強化の理由を読みやすくする
- 敗北や防衛失敗時の被害は小さめの食料減少と少数負傷に抑え、放置成長の主軸を壊さない

## 2026-06-24: 襲撃戦闘を集団交戦と死亡コストへ寄せる

### Decision

敵アリが少数で出てきて、接触しても味方が逃げるだけでは、防衛ゲームとして危機感が弱い。アリ同士の戦闘は、単純な1対1の正面押し合いではなく、複数個体が絡む fighting group、局所的な警戒、死亡を含むコストとして表現する。

### Basis

- 実験ベースのアリ戦闘モデルでは、単体個体だけでなく「2対1」などの fighting group を状態として扱っている
- アリの危険情報は短距離では警戒や突撃を生むが、長距離へ完全に共有されるとは限らず、結果として個体の犠牲が発生しうる
- アリの戦争や縄張り争いは、種や状況によって多数個体、兵隊個体、死亡・負傷個体の回収を伴う

Sources:
- https://arxiv.org/abs/1503.06094
- https://arxiv.org/abs/1306.3077
- https://arxiv.org/abs/1512.04162
- https://en.wikipedia.org/wiki/War_in_ants

### Result

- 襲撃規模を最大10匹、初期脅威ではおおむね6匹に増やした
- 敵1匹との接触は、近くのガードや働きアリが加わる集団交戦へ変更した
- 交戦中の味方は敵の正面に一列で貼り付かず、横や斜めから囲む配置にした
- 敵が勝つと味方個体が死亡し、`antPopulation` と `fallenAnts` に反映する
- 味方が十分な数で囲むと敵個体を制圧し、襲撃側の損耗として扱う
- 敵が巣周辺や餌場に居座ると、接触戦闘がなくても食料被害と低確率の死亡が発生する
- `raidState` に casualties / enemyCasualties / breachTimer を optional 追加し、旧セーブでは0へ補完する

## 2026-06-25: 交戦中の固まり感を描画で緩和

### Decision

接敵時に個体がその場で組み合う方針は維持する。ただし、静止して見えると戦闘ではなく描画停止に見えるため、戦闘中だけ脚・触角・大顎の位相アニメーションと、砂煙、衝突リング、短い噛み合いフラッシュを出して「その場で争っている」状態を表現する。

### Result

- 交戦中も味方アリと敵アリの歩行位相を進め、脚が固まらないようにした
- appendage 描画で脚、触角、大顎を状態に応じて微動させた
- `combatEffects` を保存されない一時描画として追加し、戦闘中だけ砂煙・リング・フラッシュを生成する
- エフェクトは最大96個で古いものから破棄し、リセット/終了時にも破棄する
- Playwright と verify で、交戦中の gait 進行と combat effect 発生を検証対象にした

## 2026-06-25: 戦闘後の残り方と襲撃隊形を自然化

### Decision

戦闘終了後の敵は、その場で固まったり突然消えたりさせない。制圧された敵は死体としてその場に残し、退却する敵は外周の内側ではなく画面外へ逃げる。襲撃開始時は外周円弧へ横一列に並べるのではなく、同じ進入方向を保ちながら前後差と横ずれを持つ緩い扇形で侵入させる。

### Result

- 撃破された敵は `defeated` として扱い、ライブ個体から外して死体 mesh を残す
- 退却する敵のホーム地点はワールド半径の外に置き、画面内で固まらないようにする
- 働きアリ、運搬中、帰巣中、餌場近く、巣から離れた個体を敵の攻撃対象として明示的に優先する
- 襲撃スポーン位置に前後差と横ずれを入れ、目標地点も個体ごとにずらした
- Playwright と verify で、出現位置と目標地点のばらつき、退却先が画面外であること、撃破後に死体が残ることを確認する

## 2026-06-25: 働きアリ攻撃と死体視認性を強化

### Decision

敵襲中の敵アリは、餌だけでなく、巣外で採餌・運搬・帰巣している働きアリを優先的に攻撃する。撃破された敵は、ライブ個体が消えたように見えないよう、胴体だけでなく脚と触角を含む低い死体としてその場に残す。

### Result

- 襲撃敵の働きアリ検知範囲を広げ、露出、運搬中、帰巣中、餌場近くの worker に強い優先度を付けた
- 敵死体の色、接地跡、脚、触角を強め、戦闘直後と数秒後のスクリーンショットで残存確認できるようにした
- `npm run verify:combat` を追加し、離れた worker を標的にすること、戦闘中の組み合い、戦闘後の死体残存を固定シナリオで確認する

## Current Direction
 
今後の判断では、以下を優先する。

- 放置成長の気持ちよさ
- 小さな巣が大帝国になる長期目標
- 盤面上の自律行動による観察価値
- スマホで読める UI
- 不快すぎないアリ表現
- 強化、遠征、外敵、領土がつながるゲームループ
