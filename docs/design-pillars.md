# Design Pillars

このファイルは、antEX のゲームデザイン判断がぶれないようにするための基本方針を記録する。

## Game Identity

antEX は、アリの巣を小さな群れから巨大な地下帝国へ育てていく 3D 放置ゲームである。

中心体験は、プレイヤーが細かく物を置いて反応を見ることではなく、時間経過によって巣が成長し、働きアリ、兵隊アリ、餌場、外敵、防衛が少しずつ広がっていくことにある。

## Core Fantasy

プレイヤーはアリの巣そのものを育てる。

最初は小さな巣と少数のアリだけがある。やがて餌を集め、巣を広げ、兵隊を増やし、外敵や敵対コロニーを押し返しながら、地中と地表にまたがる大帝国を作る。

## Primary Experience

- 小さなコロニーが時間で成長する
- 盤面上のアリが自律的に動く
- 餌場、地形、外敵によって採餌効率が変わる
- 強化によって成長速度、収容上限、戦闘力が上がる
- 兵隊を育て、必要なタイミングで巣から出撃させる
- 放置後に戻ると、巣が少し進んでいる

## What This Game Is Not

- 水、枝、物を置いて反応を見る実験ツールではない
- 最初から大量のアリを見せる群衆シミュレータではない
- アリを巨大に見せる観察アプリではない
- リアルタイム操作で戦闘を直接制御するアクションゲームではない

## Scale And Mood

初期状態は小さく、控えめに始める。

アリは 12 匹程度から始め、画面上の個体も小さめにする。アリが大きすぎたり、密集しすぎたりして不快に見える状態は避ける。

目標は「気持ち悪い虫の群れ」ではなく、「小さな社会が動いている」感覚である。

## Ant Visual Rules

アリはアリとして認識できるが、過度に生々しくしない。

- 頭、胸、腹の分かれたシルエットを持つ
- 腰のくびれを持つ
- 触角と 6 本脚を持つ
- ゴキブリ風の一体型楕円にしない
- 画面上では小さく、群れとして読める大きさに保つ

## World Design

マップは巣の周囲だけで完結させず、広めの自然環境として扱う。

地形、餌場、外敵、敵対アリなどを配置し、単純な盤面ではなく、採餌効率や移動経路に差が出る環境にする。

## Ecosystem Design

自然環境は複雑すぎないが、採餌の邪魔や機会を生む。

- 餌場は複数配置する
- 外敵または敵対アリは少数配置する
- 敵対アリは味方より少し大きく、茶色系にする
- 接触すると個体パラメータで押し合いが起きる
- 負けたアリは採餌行動を中断される

## Idle Growth

成長はゲームの主軸である。

食料、アリ数、収容上限、兵隊、負傷、敵脅威が時間で変化する。プレイヤーの判断は、強化、兵隊出撃、成長方向の選択に現れる。

## Combat And Defense

戦闘は 3D 盤面上で発生するが、アクションゲームのように個体を直接操作するものではない。

兵隊アリは平時は巣内に待機し、プレイヤーの出撃指示で盤面に現れる。出撃中の兵隊は巣口周辺を警戒し、敵襲時には近くの敵へ迎撃に向かう。

## Mobile First

スマホで遊べることを最優先する。

- UI は狭い画面で横にはみ出さない
- カメラはドラッグまたはタッチ中だけ動く
- hover やマウス移動だけでカメラを動かさない
- 操作量を増やしすぎない

## Technical Pillars

- Three.js は描画レイヤとして使う
- ゲーム状態は Object3D に依存させすぎない
- アリは InstancedMesh で描画する
- 初期表示は少数にする
- 描画数には上限を設ける
- GitHub Pages で公開できる軽量構成を維持する
